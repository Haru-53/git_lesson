

# Git Lesson

## リモートリポジトリとローカルリポジトリとはそれぞれ何でしょうか？

リモートリポジトリは、GitHubなどのサーバー上で管理されるリポジトリです。複数人でソースコードを共有・管理するために利用します。
ローカルリポジトリは、自分のパソコン上にあるリポジトリです。開発や編集、コミットなどの作業を行います。

## プッシュとマージの違いは何でしょうか？

プッシュ（push）は、ローカルリポジトリで行ったコミットをリモートリポジトリ（GitHub）へ送信する操作です。
マージ（merge）は、別のブランチで行った変更内容を現在のブランチへ統合する操作です。例えば、作業用ブランチの変更をmainブランチへ取り込む際に使用します。

## コミットとプッシュの違い

コミットは、ローカルリポジトリに変更内容を記録する操作です。
プッシュは、ローカルリポジトリに記録したコミットをリモートリポジトリへ反映する操作です。

## コミットのメッセージはどのように書いてあげるのが最適でしょうか？

コミットメッセージは、どのような変更を行ったのかが分かるように、簡潔で分かりやすく記述することが重要です。

## ローカルでマージするフローと、プルリクエストでマージするフローの違いは何でしょうか？

ローカルでマージするフローは、自分のパソコン上でブランチをマージし、その結果をリモートリポジトリへプッシュする方法です。
プルリクエストでマージするフローは、変更内容をGitHub上でレビューしてもらい、問題がなければ承認後にマージする方法です。


## コンフリクトを起こしてしまった場合、どう対処すべきですか？

コンフリクトが発生した場合は、まず競合している箇所を確認し、どちらの変更内容を採用するか、または両方を取り込むかを判断します。
チーム開発では、自分だけで判断せず、変更を行ったメンバーと相談しながら修正内容を決定し、コンフリクトを解消してからマージを行います。
